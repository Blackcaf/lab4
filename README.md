Проект реализует стековый язык `Forth`, собственную систему команд, микрокомандную модель процессора, транслятор и симулятор.

## Архитектура проекта

Цепочка работы выглядит так:

`Forth-программа` → `translator.py` → `машинный код (.bin + .txt)` → `machine.py` → `журнал работы и вывод`

### Что входит в текущую реализацию

- язык `Forth` с процедурами, переменными, числами, строками и символами;
- execution token: `' word` и `EXECUTE`;
- ветвления `IF/ELSE/THEN`, циклы `BEGIN/WHILE/REPEAT` и `BEGIN/UNTIL`;
- арифметика, логика, сравнения, стековые операции;
- port-mapped I/O (`KEY`, `EMIT`);
- строковый литерал `S"..."` в формате P-string;
- гарвардская память, `I-CACHE`/`D-CACHE`, тактовая модель (`tick`);
- golden-тесты для end-to-end проверки.

## Язык программирования `Forth`

Реализован Forth-подобный стековый язык. Программа читается слева направо, а все данные передаются через стек данных.

### Синтаксис

```ebnf
<программа>              ::= { <строка> }
<строка>                 ::= <определение_слова>
                         | <определение_переменной>
                         | <исполняемый_токен> { <исполняемый_токен> }
                         | <комментарий>
                         | <пустая_строка>

<комментарий>            ::= '\' <любые_символы_до_конца_строки> '\n'
<пустая_строка>          ::= <пробел>* '\n'

<определение_слова>      ::= ':' <имя_слова> { <токен> } ';'
<определение_переменной> ::= ('VARIABLE' | 'CREATE') <имя_переменной> ['ALLOT' <число>]

<токен>                  ::= <число>
                         | <символ_литерал>
                         | <строка_литерал>
                         | <имя_из_словаря_слов>
                         | <управляющая_структура_FORTH>
                         | <любое_другое_слово>

<имя_слова>              ::= <идентификатор>
<имя_переменной>         ::= <идентификатор>
<идентификатор>          ::= ( <буква> | <цифра> | '-' | '_' | '<' | '>' | '@' | '!' | '+' | '*' | '=' | '/' )+

<число>                  ::= [ '-' ] <десятичное_число>
                         | '0x' <шестнадцатеричное_число>
                         | '0b' <двоичное_число>
<десятичное_число>       ::= <цифра>+
<шестнадцатеричное_число>::= ( <цифра> | 'A'..'F' | 'a'..'f' )+
<двоичное_число>         ::= ('0' | '1')+
<цифра>                  ::= '0'..'9'
<буква>                  ::= 'a'..'z' | 'A'..'Z'

<символ_литерал>         ::= '[CHAR]' <любой_символ_без_пробела>
<строка_литерал>         ::= 'S"' <содержимое_строки> '"'
<управляющая_структура_FORTH> ::= 'IF' | 'ELSE' | 'THEN' | 'BEGIN' | 'WHILE' | 'REPEAT' | 'UNTIL'
```

### Семантика

- **Стек данных** используется для аргументов и результатов всех операций.
- **Слова** задаются через `: имя ... ;` и могут вызывать друг друга.
- **Execution token**: `' word` кладет адрес слова на стек, `EXECUTE` вызывает слово по этому адресу.
- **Переменные** создаются через `VARIABLE` или `CREATE ... ALLOT` и размещаются в статической области памяти данных.
- **Литералы** (`42`, `0xABC`, `0b101`, `[CHAR] A`, `S"hello"`) помещаются на стек или в секцию данных.
- **Строки** хранятся как P-string: сначала длина, затем символы по одному слову.
- **Ввод/вывод**: `KEY` читает символ из порта `0`, `EMIT` выводит символ через порт `1`.
- **Потоковый ввод (`stream`)**: при исчерпании входного буфера моделирование останавливается.
- **Управление потоком** реализовано через условные и безусловные переходы, с поддержкой циклов и ветвлений.

## Организация памяти

Используется **гарвардская архитектура**: память команд и память данных разделены.

- **Машинное слово** - 32 бита.
- **Память команд** - 64K слов, отдельное адресное пространство.
- **Память данных** - 64K слов, отдельное адресное пространство.
- **Кеши** - `I-CACHE` и `D-CACHE`, direct-mapped, с задержкой **1 такт при hit** и **10 тактов при miss**.
- **I/O** - port-mapped, порты не занимают адреса памяти данных.

### Раскладка памяти данных

```text
0x0000 ───────────────────────────────
         статические данные
         - переменные
         - строковые литералы
         свободная область
         стек данных (data_sp)
         стек вызовов (SP)
0xFFFF ───────────────────────────────
```

## Система команд (RISC)

Процессор использует фиксированный 32-битный формат инструкций.

### Регистры

- `ZERO` (0) - всегда 0;
- `SP` (1) - стек вызовов;
- `RA` (2) - адрес возврата;
- `T0`–`T3` (3–6) - временные регистры;
- `A1` (7) - регистр аргумента/результата.

Дополнительно в симуляторе используются внутренние регистры `PC`, `MAR`, `MDR`, `IR_REG` и `data_sp`.

### Opcode и форматы

#### Системные инструкции

- `NOP (0x00)`
- `HALT (0x01)`

#### Арифметика и логика, R-type

- `ADD`, `SUB`, `MUL`, `DIV`, `MOD`
- `AND`, `OR`, `XOR`
- `CMP`, `SHL`, `SHR`

Формат: `[Opcode:6][rs:5][rt:5][rd:5][unused:11]`

#### Инструкции с непосредственным значением, I-type

- `ADDI`, `LOAD`, `STORE`
- `JZ`, `JNZ`
- `LUI`, `ORI`
- `IN`, `OUT`

Формат: `[Opcode:6][rs:5][rt:5][imm:16]`

#### Работа со стеком

- `PUSH`
- `POP`

#### Переходы и вызовы

- `JMP`
- `CALL`
- `RET`

### Кодирование

Старшие 6 бит инструкции кодируют opcode. Остальные поля зависят от типа инструкции.

```text
31...26 | 25...0
Opcode  | параметры инструкции
```

## Микрокоманды и Control Unit

Каждая инструкция выполняется как последовательность микрокоманд (`MicroOp`) по тактам.

### Fetch cycle

1. `LATCH_MAR_PC`
2. `INSTR_READ`
3. `LATCH_PC_INC`
4. `LATCH_IR`

После выборки выполняется микропрограмма конкретной инструкции.

### Что моделируется

- лачинг регистров и шин;
- операции АЛУ;
- чтение/запись памяти через кеши;
- `PORT_READ` / `PORT_WRITE`;
- переходы, вызовы и возвраты;
- завершение инструкции и останов процессора.

## АЛУ

В симуляторе АЛУ поддерживает:

- сложение, вычитание, умножение;
- целочисленное деление и остаток;
- битовые операции;
- сравнение;
- сдвиги;
- загрузку старших 16 бит (`LUI`).

## Модуль условий

Сравнения Forth (`=`, `<>`, `<`, `>`, `<=`, `>=`, `0=`, `0<>`) реализованы через комбинацию инструкций ISA и стековых подпрограмм.

## Ввод/вывод

I/O выполнен через отдельные порты:

- `port 0` - вход;
- `port 1` - выход.

`KEY` и `EMIT` транслируются в соответствующие `IN`/`OUT`-последовательности.

## Транслятор

CLI:

```bash
python translator.py <source_file.f> <target_file.bin>
```

Транслятор работает в несколько проходов:

1. разбирает исходный текст;
2. строит словарь встроенных и пользовательских слов;
3. генерирует код и данные;
4. разрешает релокации для строк и переменных;
5. пишет бинарный файл `.bin` и листинг `.txt`.

## Модель процессора

CLI:

```bash
python machine.py <binary_code_file> <input_file>
```

Симулятор:

- читает машинный код из бинарного файла;
- читает вход из текстового файла;
- исполняет программу такт за тактом;
- выводит результат и число тактов;
- моделирует кеши и port-mapped I/O.

## Тестирование

Корректность проверяется интеграционными golden-тестами на базе `pytest`.

### Набор golden-тестов

1. `hello.yml` - вывод `Hello, World!`.
2. `cat.yml` - echo-программа, читает ввод и печатает его обратно.
3. `hello_user_name.yml` - приветствие по введённому имени.
4. `sort.yml` - сортировка списка чисел.
5. `double_add.yml` - демонстрация 64-битной арифметики.
6. `euler6.yml` - алгоритм согласно варианту / демонстрация арифметики.
7. `cache_test.yml` - проверка влияния кеш-памяти на число тактов.

### Запуск тестов

```bash
pytest
```

Или через `Makefile`:

```bash
make lint
make typecheck
make test
make coverage
make check
```

В репозитории настроен CI (`.github/workflows/ci.yml`): автоматически запускаются `ruff`, `mypy` и `pytest`.
